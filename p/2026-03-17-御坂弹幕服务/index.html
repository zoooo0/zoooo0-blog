Docker 部署御坂网络弹幕服务 Misaka Danmu Server

2026-03-17 技术 Docker 3,524

什么是御坂网络弹幕服务

御坂网络弹幕服务（Misaka Danmu Server）是一个功能强大的自托管弹幕聚合与管理服务，兼容 dandanplay API 规范。它能够从多个视频平台实时获取弹幕数据，为你的私人媒体库（如 Emby、Jellyfin、Plex）提供弹幕支持。

GitHub: l429609201/misaka_danmu_server

核心功能

  • 智能匹配：通过文件名或元数据（TMDB、TVDB 等）自动匹配媒体内容
  • Web 管理界面：提供直观的 UI 用于搜索/导入弹幕、管理媒体源、创建 API Token、配置搜索优先级
  • 弹幕源管理：支持从资源仓库安装或上传离线弹幕源包，具备备份/恢复功能
  • 元数据整合：整合 TMDB、TVDB、Bangumi、豆瓣、IMDb 等多个元数据源
  • 自动化支持：支持 Sonarr、Radarr、Emby 的 Webhook 通知，实现弹幕自动导入
  • 多播放器兼容：支持 SenPlayer、Infuse、小幻、yamby、eplayerx 等播放器

与 danmu_api 对比

功能 danmu_api misaka_danmu_server
基本弹幕抓取 ✅ ✅
dandanplay API 兼容 ✅ ✅
Web 管理界面 ❌ ✅
媒体库/元数据匹配 ❌ ✅ TMDB/TVDB/Bangumi/豆瓣/IMDb
Webhook 自动化 ❌ ✅ Sonarr/Radarr/Emby
弹幕源管理 有限 ✅ 完整的源管理系统
Token 授权体系 简单 ✅ 完整权限控制
部署复杂度 简单（无需数据库） 中等（需要 MySQL）
适用场景 个人轻量使用 完整媒体库管理

准备工作

服务器要求

  • 内存：建议 2GB 以上
  • 磁盘：建议 20GB 以上
  • 网络：建议使用中国大陆 IP，海外 IP 可能因网络或版权限制导致部分弹幕源不可用

安装 Docker

curl -fsSL https://get.docker.com | sh

Docker Compose 部署

创建项目目录

mkdir -p ~/misaka-danmu && cd ~/misaka-danmu

创建 docker-compose.yml

version: "3.8"
services:
  mysql:
    image: mysql:8.1.0-oracle
    container_name: danmu-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: "your_strong_root_password"
      MYSQL_DATABASE: "danmuapi"
      MYSQL_USER: "danmuapi"
      MYSQL_PASSWORD: "your_strong_user_password"
      TZ: "Asia/Shanghai"
    volumes:
      - ./db-data:/var/lib/mysql
    command:
      - '--character-set-server=utf8mb4'
      - '--collation-server=utf8mb4_unicode_ci'
      - '--binlog_expire_logs_seconds=259200'
      - '--default-authentication-plugin=mysql_native_password'
    healthcheck:
      test: ["CMD-SHELL", "mysqladmin ping -u$${MYSQL_USER} -p$${MYSQL_PASSWORD}"]
      interval: 5s
      timeout: 3s
      retries: 5
      start_period: 30s
    networks:
      - misaka-net

  danmu-app:
    image: l429609201/misaka_danmu_server:latest
    container_name: misaka-danmu-server
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      - PUID=1000
      - PGID=1000
      - UMASK=0022
      - TZ=Asia/Shanghai
      - DANMUAPI_DATABASE__TYPE=mysql
      - DANMUAPI_DATABASE__HOST=mysql
      - DANMUAPI_DATABASE__PORT=3306
      - DANMUAPI_DATABASE__NAME=danmuapi
      - DANMUAPI_DATABASE__USER=danmuapi
      - DANMUAPI_DATABASE__PASSWORD=your_strong_user_password
      - DANMUAPI_ADMIN__INITIAL_USER=admin
    volumes:
      - ./config:/app/config
    ports:
      - "7768:7768"
    networks:
      - misaka-net

networks:
  misaka-net:
    driver: bridge

启动服务

docker compose up -d

查看启动日志：

docker compose logs -f danmu

首次启动会显示初始管理员密码，请记录下来。

初始配置

访问 Web 管理界面

浏览器访问 http://你的服务器IP:7768

使用管理员账户登录： - 用户名：admin - 密码：启动日志中显示的初始密码

弹幕源配置

  1. 进入 系统设置 > 弹幕源管理
  2. 点击 从仓库安装 获取最新的弹幕源
  3. 根据需要启用或禁用特定源
  4. 可以调整源的优先级

Token 管理

  1. 进入 系统设置 > Token 管理
  2. 点击 新建 Token
  3. 填写备注并保存
  4. 复制生成的 Token 用于播放器配置

播放器配置

SenPlayer

  1. 打开 SenPlayer，进入 设置 > 弹幕设置 > 自定义弹幕 API
  2. 填入 API 地址：http://你的服务器IP:7768/{Token}

Infuse

  1. 进入 设置 > 元数据 > 弹幕
  2. 选择 自定义
  3. 填入 API 地址：http://你的服务器IP:7768/{Token}

小幻播放器

需要在 API 地址后面加上 /api/v2：

http://你的服务器IP:7768/{Token}/api/v2

Yamby (Android)

填入 API 地址：http://你的服务器IP:7768/api/v1/{Token}

Hills (Android/Windows)

填入 API 地址：http://你的服务器IP:7768/api/v1/{Token}

高级配置

Webhook 自动化

支持 Emby、Jellyfin、Plex、Sonarr、Radarr 的 Webhook 通知，实现弹幕自动导入。

  • Emby Webhook URL: http://你的服务器IP:7768/webhook/emby
  • Sonarr Webhook URL: http://你的服务器IP:7768/webhook/sonarr
  • Radarr Webhook URL: http://你的服务器IP:7768/webhook/radarr

元数据源配置

支持 TMDB、TVDB、Bangumi、豆瓣、IMDb 等元数据源。

优先级建议： - 通用：TMDB > TVDB > Bangumi > 豆瓣 > IMDb - 动画为主：Bangumi > TMDB > TVDB - 华语内容：豆瓣 > TMDB > IMDb

AI 功能配置

支持 DeepSeek、SiliconFlow、OpenAI、Google Gemini 等 AI 提供商，增强匹配能力。

更新服务

cd ~/misaka-danmu
docker compose pull
docker compose up -d

注意事项

  • 建议使用中国大陆 IP 部署
  • 请合理使用，避免滥用导致 IP 被封禁
